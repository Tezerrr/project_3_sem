Документация бэкенд части

Проектная документация описывает бэкенд часть приложения, которая включает в себя 7 методов, связанных с регистрацией/авторизацией пользователей, работой с нейронной сетью и обновлением JWT-токена.
Методы регистрации/авторизации пользователей
1. /register

    Метод: POST
    Параметры входных данных: JSON-файл с полями username, email, password.
    Описание: Регистрирует нового пользователя, добавляя данные в таблицу пользователей. При успешной регистрации возвращает JWT-токены.

2. /login

    Метод: POST
    Параметры входных данных: JSON-файл с полями email, password.
    Описание: Авторизует пользователя, возвращая JWT-токены и имя пользователя при успешной авторизации.

3. /delete

    Метод: DELETE
    Параметры входных данных: JWT-токен в запросе.
    Описание: Удаляет данные пользователя из базы данных при наличии его в системе.

Метод обновления токенов
/refresh

    Метод: POST
    Параметры входных данных: Только refresh-токен.
    Описание: Обновляет пару токенов и возвращает их.

Методы работы с нейронной сетью
1. /recognition

    Метод: POST
    Параметры входных данных: JWT-токен, файл формата, указанного в allowed_extensions.
    Описание: Загружает файл, добавляет данные в базу данных, запускает нейронную сеть и возвращает результат работы в формате JSON.

2. /history

    Метод: GET
    Параметры входных данных: JWT-токен.
    Описание: Возвращает историю запросов пользователя по его ID.

3. /translation

    Метод: POST
    Параметры входных данных: JSON-файл с полями text, source, res.
    Описание: Проводит перевод текста и возвращает результат.

SQL-таблицы
1. users

    Описание: Таблица, где хранятся данные пользователей для входа.

2. images

    Описание: Таблица, где хранятся данные по истории запросов.

Примечания

    Для работы с файлами используется формат, указанный в файле allowed_extensions.
    Проект использует две SQL-таблицы: users и images.

Эта документация предоставляет общее понимание методов и структуры бэкенд части проекта
