Документация бэкенд части<br>
Проект содержит всего 7 методов. Из них 3 метода относятся к регистрации/авторизации пользователя, 3 работают с нейронной сетью и результатами её работы и 1 для обновления jwt-токена<br>
Регистрация/авторизация пользователей:<br>
  1)/register-метод принимает json-файл с полями username, email, password. При успешном считывании данных мы добавляем данные пользователя в таблицу и возвращаем jwt-токены<br>
  2)/login-метод принимает json-файл с полями email,password. При успешном выполнении авторизует пользователя и возвращает jwt-токены и username<br>
  3)/delete-данный метод работает при подаче jwt-токена в запросе. Он удаляет данные пользователя при наличии его в базе данных<br>
Обновление токенов-/refresh-метод принимает только refersh-токен и возвращает обновленную пару токенов<br>
Работа с нейронной сетью:<br>
  1)/recognition-данный метод работает при подаче jwt-токена. Принимает файл формата, указанного в файле allowed_extensions, добавляет в базу данных id пользователя, путь к загруженному файлу и результат работы. Выводит результат работы в json-формате<br>
  2)/history-данный метод работает при подаче jwt-токена. Выдаёт историю запросов пользователя по его id<br>
  3)/translation-метод принимает json-файл с полями text,source,res. Выдаёт перевод<br>
Также используется две sql-таблицы:users, images<br>
users-таблица в которой хранятся данные пользователей для входа<br>
images-таблица в которой хранятся данные по истории запросов<br>
